Graph Modeling Choice

Labeled Property Graph (LPG) vs Resource Description Framework (RDF)

LPG Neo4j / openCypher Style
Nodes with Labels
Labels categorize what a node is. A node can have multiple labels.
(:Drug {name: "Pembrolizumab"})
Relationship Types
Types describe what the connection means. Each relationship has exactly one type.
-[:TREATS]->
Properties on Nodes
Key-value pairs stored directly on any node.
{mechanism: "PD-1 inhibitor", year: 2014}
Properties on Relationships
Key-value pairs stored directly on any relationship — a major LPG advantage.
-[:TREATS {efficacy: 0.85, source: "Phase 3"}]->
Query Language
Cypher (Neo4j) / openCypher (Neptune) / Gremlin (TinkerPop)
Vector Search
Native HNSW index on node properties. Vectors are first-class properties.
CALL db.index.vector.queryNodes()

Best For

  • GraphRAG (vector + graph in one query)
  • Path traversal and recommendations
  • Operational / real-time applications
  • Edge-weighted analytics (properties on relationships)
  • Developer-friendly data modeling
SUPPORTED BY Neo4j Aura • Neptune Analytics (openCypher) • Neptune DB (Gremlin / openCypher)
RDF W3C / SPARQL Style
Triples
Everything is Subject → Predicate → Object. One fact = one triple.
:Drug123 :treats :Disease456 .
Classes via rdf:type
No labels. Typing is done through rdf:type triples linking to ontology classes.
:Drug123 rdf:type :Drug .
Properties as Triples
Each property is a separate triple. No key-value pairs on nodes.
:Drug123 :drugName "Pembrolizumab" .
No Native Edge Properties
Requires reification, named graphs, or RDF-star to attach metadata to a relationship.
:Treatment001 :efficacy "0.85" .
(extra modeling required)
Query Language
SPARQL 1.1 with federation (SERVICE clause queries remote endpoints).
Schema & Reasoning
RDFS/OWL ontologies enable inference. SHACL validates graph structure.
OWL infers: targets ImmuneCheckpoint ⇒ isImmunotherapy

Best For

  • OWL / RDFS formal ontology and reasoning
  • W3C semantic standards compliance
  • Cross-organization interoperability (global URIs)
  • SPARQL federation across endpoints
  • SHACL validation and data governance
SUPPORTED BY Neptune DB (SPARQL) • Apache Jena • Blazegraph • GraphDB • Stardog
⚠ No native vector search in RDF — requires a separate vector layer (e.g., OpenSearch kNN)
Key Difference
LPG:  Labels on nodes, Types on relationships, Properties on both
RDF:  No labels, no types, no edge properties — just triples
